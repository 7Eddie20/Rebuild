# 📁 Topic: Introduction

## ✅ Concepts Practiced

1. Basic input using `cin`
2. Output using `cout`
3. Taking full-line input with `getline()`
4. Resolving conflicts with `cin.ignore()`

## 🗂️ Projects in This Folder

- `Add2Numbers` – Simple program to add two integers, practicing input/output
- `bookRating` – Uses `getline()` and `cin.ignore()` to capture book reviews
- `CinIgnore` – Demonstrates how `cin.ignore()` prevents input issues
- `contactForm` – Captures full name and details using `getline()`
- `stringInput` – More practice with line-based input

## 🐛 Bugs or Confusions Faced

- Placement of `cin.ignore()` after `cin` is crucial
- Forgetting `cin.ignore()` causes `getline()` to be skipped

## 💡 Key Learnings

- `getline()` is used to take full-line input including spaces
- Always use `cin.ignore()` after `cin` and before `getline()`
- `'cin >>'` is for token-based input, `'cout <<'` is for output
- Input flows need attention to whitespace and buffer handling
